# 📓 Lessons Learned: Ajou LLM Agent 변환 작업

## 🎨 스킨 및 디자인 (Skin Engine)
- **전용 스킨 자동 활성화**: 특정 프로바이더(예: `ajoullm`)를 사용할 때 그에 맞는 테마(`ajou`)를 자동으로 활성화하면 사용자 경험이 크게 향상됨.
- **ASCII 아트의 힘**: CLI 환경에서 브랜드 아이덴티티를 나타내는 가장 효과적인 방법은 색상이 적용된 ASCII 로고와 히어로 아트임.
- **스피너 커스터마이징**: `thinking_verbs`와 `waiting_faces`를 도메인(학업, 연구)에 맞게 조정하여 에이전트의 페르소나를 강화함.

## 🔌 프로바이더 및 모델 통합
- **중앙화된 프로바이더 정의**: `hermes_cli/providers.py`에서 `HERMES_OVERLAYS`를 통해 프로바이더를 정의하면 CLI 전체에서 일관되게 인식됨.
- **별칭(Aliases) 활용**: `ajou`, `ajou-llm` 등 다양한 별칭을 등록하여 사용자가 편리하게 접근할 수 있도록 함.
- **동적 모델 카탈로그**: `_PROVIDER_MODELS`에 기본 모델들을 등록해두면 사용자가 별도 설정 없이 바로 선택 가능함.

## 🛠️ 셋업 및 에러 핸들링
- **셋업 마법사 우선순위**: 셋업 시작 시 타겟 사용자(아주대 학생)를 위한 퀵 셋업 옵션을 제공하여 복잡한 과정을 단축함.
- **가이드 중심의 에러 메시지**: API 에러 발생 시 단순 에러 코드만 보여주는 것이 아니라, 해결을 위한 가이드 링크를 함께 제공하여 자기 해결(Self-service)을 유도함.

## 🧠 프로젝트 관리
- **G-Plan 준수**: `tasks/todo.md`를 통해 계획을 세우고 단계별로 실행하는 것이 복잡한 변환 작업에서 길을 잃지 않는 핵심임.
- **Obsidian 동기화**: 세션의 결과를 외부 메모리에 기록하여 지식의 영속성을 유지함.
